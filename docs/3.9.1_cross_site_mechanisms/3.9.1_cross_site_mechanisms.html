Introduction

This exercise covers two important vulnerabilities: Cross Site Scripting and Cross Site Request Forgery. Both of these fall into the category of "Cross Site Mechanisms" because they abuse vulnerabilities in websites to target a user's browser. In the case of Cross Site Scripting, an attacker is able to execute scripts in the victim's browser. With Cross Site Request Forgery, an attacker can trick a user's browser into sending an authenticated request crafted by the attacker. These are called cross site mechanisms because they almost always involve an adversarial web site and the vulnerable web site. As you will see, simply clicking a link on an attacker's webpage or even just visiting one can trigger this attack.
