Command-Line Text Editors

In many of these exercises, we interact with programs exclusively through the command line. When it comes time to edit a source file or read an output file, it is cumbersome to switch attention to a GUI and navigate to the current directory just to open a file. Instead, we use a command-line text editor to interact with text files without leaving the command line shell. Before the widespread adoption of GUI environments, these text editors were the only option. Nearly all distributions of Linux still include at least one of these editors by default.

Nano

If you've never used a command-line text editor and don't plan on using one often, Nano is likely your best choice. It is great for small edits and fast changes. It is included in many Linux distributions (including the security exercises virtual machine).

To read or edit a file named myFile using nano, simply enter the command nano myFile in the shell. Commands are listed at the bottom in the form of ^X Exit. This means press the ctrl key and the x key simultaneously to exit the file. It will ask if you want to save your modifications (if you made any).

Vim

Another simple text editor to consider is Vim, which is included with nearly every Linux distribution. This is a slightly more powerful/customizable text-editor compared to Nano. Note that it is a "mode-based" editor, so you will have different commands/options depending on your current mode.

Similar to Nano, you can open a file named myFile by typing vim myFile. First press the i key to switch into "insert mode". Now you can navigate the document using the arrow keys and make changes to the code. When you are done editing, press the esc key to leave "insert mode", then type :wq to "write and quit" (i.e., "save and exit"). If you don't wish to save, type :q! instead of :wq.
